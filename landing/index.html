Chapter one

How it works

Four stages, all running on your machine, from key-down to injected text in roughly one second.

01
Capture
cpal → rubato → 16kHz mono

Hold the hotkey. The mic is captured at its native rate and resampled down to what the speech model expects.

02
Transcribe
Parakeet TDT 0.6B v2

An on-device speech recognizer (int8 ONNX) turns audio into raw text — no network call, ever.

03
Clean up
Qwen2.5-1.5B via llama.cpp

A small local language model strips fillers and self-corrections and fixes punctuation — one deterministic pass.

04
Place
AX insert → clipboard fallback

Release the key. The cleaned text is typed into whatever app has focus, with a clipboard-paste fallback.

Chapter three

Architecturally verifiable, not a promise

The claim isn't "we don't look" — it's that the audio never has a path to leave. The source is open; you can read the injection and capture code yourself.

Cloud dictation tools
  • – Voice streamed to remote servers
  • – Some capture periodic screenshots of your active window
  • – Retain data by default
  • – Monthly subscription
  • – Privacy is a policy you have to trust
suzune
  • + Audio never leaves the device
  • + No screenshots, no screen context of any kind
  • + No telemetry, no account, no upload
  • + Free, MIT licensed
  • + Privacy is source code you can read

suzune asks for exactly two macOS permissions — Microphone and Accessibility — and both stay on your machine. A network-permission-free build is the proof, not the pitch.
